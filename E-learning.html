JavaScript Fundamentals › Module 2 — Core Concepts
Variables & Data Types

In JavaScript, variables are named containers that store data values. Modern JavaScript uses let and const — the older var is best avoided in new code.

Declaring variables

Use const by default. Switch to let only when you know the value will change.

// const — value cannot be reassigned const name = "Alice"; const age = 28; // let — value can change let score = 0; score = 42; // ✓ allowed JS

The 6 primitive types

Every value in JavaScript belongs to a type. Primitives are the simplest building blocks.

const text = "hello"; // string const count = 42; // number const pi = 3.14; // number (no float type) const active = true; // boolean const nothing = null; // null — intentional absence let ghost; // undefined — not yet set JS
💡 Tip

Use typeof to inspect a value's type at runtime: typeof "hello" returns "string". This is handy when debugging.

Type coercion — watch out!

JavaScript will silently convert types when you mix them in operations. This catches many beginners off guard.

console.log("5" + 3); // "53" (string wins in +) console.log("5" - 3); // 2 (number for - * /) console.log(0 == ""); // true (loose equality) console.log(0 === ""); // false (strict — always use this) JS
